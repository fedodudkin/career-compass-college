## Возможности

- **4 специальности** с детальными дорожными картами навыков
- **Отслеживание прогресса** — отмечай пройденные темы, прогресс сохраняется в браузере
- **Данные о зарплатах** — Junior / Middle / Senior с прогнозом до 2028 года
- **Работодатели** — ссылки на карьерные страницы российских IT-компаний
- **Форма обратной связи** — с валидацией, счётчиком символов и сохранением в localStorage
- **Тёмная тема** — единственная, адаптирована под все экраны
- **Анимации появления** — Intersection Observer, однократно, без повторений
- **Доступность** — семантическая разметка, ARIA-атрибуты, навигация с клавиатуры

## Запуск

1. Открыть папку `src/` через **Live Server** (расширение VS Code или любой HTTP-сервер)
2. Перейти на `http://127.0.0.1:5500/`

> ⚠️ **Важно:** открывать через HTTP-сервер, не через `file://`.  
> Это необходимо для ES-модулей и `fetch('data/data.json')`.

**Быстрый запуск через Python:**
```bash
cd src
python -m http.server 5500
```

## Страницы

### Главная (`index.html`)
Сетка из 4 карточек специальностей. Каждая карточка показывает:
- PNG-иконку специальности
- Код и название специальности
- Краткое описание (до 3 строк, `line-clamp`)
- Бейдж актуальности («Высокий спрос», «Топ направление» и т.д.)
- Счётчик навыков
- Прогресс-бар пройденных навыков

Клик по карточке → `roadmap.html?spec=web` (нужная специальность активируется автоматически и страница прокручивается к ней).

### Roadmap (`roadmap.html`)
Основная страница с дорожными картами. Возможности:
- Фильтрация по специальности (кнопки-фильтры с PNG-иконками)
- Сворачивание / разворачивание секций специальностей
- Отметка навыков пройденными (кнопка-галочка на карточке или кнопка в модалке)
- Модалка навыка: подробное описание, список тем для изучения
- Модалка зарплат: таблица Junior/Middle/Senior, прогноз, список работодателей
- Глобальный счётчик прогресса (`X / Y навыков пройдено`)
- Информационный disclaimer о самостоятельном подборе учебных материалов

### О сервисе (`about.html`)
Информационные карточки с описанием ключевых возможностей сервиса.

### Заявка (`form.html`)
Форма обратной связи:
- Валидация полей в реальном времени (при потере фокуса и при вводе)
- Счётчик символов в поле «Сообщение» (максимум 500)
- Сохранение заявки в `localStorage` (ключ `form_submissions`)
- Анимированное уведомление об успехе с автоскрытием через 6 секунд

## Архитектура JS

```
main.js ──import──▶ reveal.js     (side-effect: инициализирует __revealObserve)

index.js   ──import──▶ utils.js   (loadSpecialtiesData, getDoneSkills)
roadmap.js ──import──▶ utils.js   (+ toggleDoneSkill, isSectionCollapsed, formatCurrency)
form.js                            (автономный IIFE, без внешних зависимостей)
```

### Ключевые архитектурные решения

**`form.js` — автономный IIFE**  
Не использует `import`, чтобы исключить сбои из-за цепочки ES-модулей.  
Реализует `saveSubmission()` прямо внутри файла через `localStorage`.

**`quickToggle()` — хирургический DOM-патч**  
При отметке навыка НЕ вызывает `renderAll()` (это вызвало бы повторную анимацию).  
Точечно обновляет только нужные узлы: классы карточки, кнопку-галочку, счётчик секции.

**`fetch('data/data.json')` — правильный путь**  
Путь резолвится относительно HTML-страницы, а не JS-файла.  
`data/data.json` (не `../data/data.json`) — работает при любом расположении сервера.

**URL-навигация Главная → Roadmap**  
`index.js` передаёт `?spec=web` в URL при клике на карточку.  
`roadmap.js` читает параметр через `URLSearchParams`, активирует фильтр до `renderAll()`  
и после отрисовки скроллит к нужной секции через `scrollIntoView`.

**`globalThis` вместо `window`**  
Используется в `main.js`, `index.js`, `roadmap.js` для совместимости  
с современными стандартами (работает в браузере, Node.js и Web Workers).

## Система анимаций

`reveal.js` — универсальная система появления элементов на всех страницах.

**Использование:** добавь класс `animate-on-reveal` к любому элементу:
```html
<div class="animate-on-reveal">Этот блок плавно появится при скролле</div>
```

**Принцип работы:**
1. Начальное состояние — невидим (`opacity: 0`), смещён вниз на `20px`
2. `IntersectionObserver` добавляет `.is-visible` при входе в viewport (`threshold: 7%`)
3. CSS-transition: только `opacity` + `translateY`, `0.5s`, `cubic-bezier(0.22, 1, 0.36, 1)`
4. После добавления `.is-visible` → `observer.unobserve()` — гарантия однократности
5. Stagger-задержка: 100ms × порядковый номер среди соседних элементов

**Настройка задержки вручную:**
```html
<div class="animate-on-reveal" style="--reveal-delay: 200ms">...</div>
```

**`prefers-reduced-motion`:** все элементы показываются мгновенно без анимации.

**Без конфликтов с hover:** `transition` в `.animate-on-reveal` затрагивает только  
`opacity` и `translateY` — hover-трансформации карточек (`scale`, `translateY`) не пересекаются.

## Доступность A11y

| Элемент | Решение |
|---------|---------|
| Навигация | `<nav aria-label="Навигация">` (хлебные крошки) |
| Мобильное меню | `aria-expanded`, `aria-controls` на бургере |
| Карточки специальностей | `<article role="button">`, `aria-label`, навигация Enter/Пробел |
| Прогресс-бар | `role="progressbar"`, `aria-valuenow/min/max`, `aria-label` |
| Модальные окна | `role="dialog"`, `aria-modal`, `aria-labelledby` |
| Форма | `aria-invalid`, `aria-describedby` при ошибках, `aria-live` на ошибках |
| Уведомление формы | `role="status"`, `aria-live="polite"` |
| Иконки | `aria-hidden="true"` на декоративных, `alt` на информативных |
| Внешние ссылки | `rel="noopener noreferrer"` + `target="_blank"` |
| Анимации | Отключаются при `prefers-reduced-motion: reduce` |

## Хранение данных

Все данные хранятся **только в браузере пользователя**. Никаких серверов и регистрации.

| Ключ `localStorage` | Тип | Содержимое |
|--------------------|-----|-----------|
| `roadmap_done` | `Object` | `{ [skillId: string]: true }` — пройденные навыки |
| `collapsed_<specId>` | `string` | `"1"` — свёрнутые секции специальностей |
| `form_submissions` | `Array` | Список отправленных заявок с датой |

**Очистка данных:** DevTools → Application → Local Storage → удалить нужные ключи.
